# 変更仕様書
Date: 2026-04-08

## 背景と目的

`design-concept.md` の方針変更に合わせ、以下の新方針を全ドキュメント・実装・テストに反映する。

**新方針（design-concept.md より）：**
- `detach()` と `attach()` を除く各 API は、内部で必ず `validate()` を呼ぶ
- `detach()` と `attach()` を除く各 API は `rows` と `table` のどちらを受け取ってもよい
- `attach()` は `validate()` を呼ばない低レベルプリミティブ。受け取った `rows` と `meta` をそのまま組み立てる
- すでに条件を満たす `table`（`attrs` 付き）を `$cols` なしで `validate()` に渡すと、同一参照をそのまま返す
- `detach()` を除く API の出力は常に `table`

## 1. `src/TableTools.pm` の変更

### 1-1. 冒頭コメント（Terms + Rules）を全面更新する

`package` 宣言直下のコメントを `design-concept.md` の Rules セクションと完全一致させる。

```perl
# Terms:
# AoH はハッシュリファレンスの配列リファレンス
# rows はメタデータを持たない AoH
# table は先頭行にメタデータを持つ AoH
# table のメタデータは '#' に置く
# attrs はカラム名をキーに持つハッシュで、値は num または str
# order はカラム名の並びを表す配列リファレンス
#
# Rules:
# detach() と attach() を除く API は rows でも table でも受け取れる
# detach() を除く API の出力は table とする
# attach() は validate() を呼ばず、rows と meta から table を組み立てる
# attrs は必須で、order は列順を指定した validate() のときだけ付く
# orderby() は attrs に従って num は数値、str は文字列として並べる
# group() は入力順をそのまま使うので、必要なら先に orderby() を使う
# group() では非連続な同一キーの再出現をエラーにする
# expand() は group() 済みの table を平坦化して table を返す
```

### 1-2. `validate` にアーリーリターンを追加する

関数冒頭に以下の条件を追加する。`$cols` が指定されておらず、かつ入力の先頭行が `attrs` を持つ場合は、同一参照をそのまま返す。

```perl
sub validate {
    my ($aoh, $cols) = @_;
    if (!$cols && @$aoh && exists $aoh->[0]{'#'} && $aoh->[0]{'#'}{attrs}) {
        return $aoh;    # 同一参照を返す（再検証しない）
    }
    ...
}
```

**実装上の判断：** アーリーリターン条件は「先頭行に `attrs` が存在する」のみを確認する。各データ行が `attrs` の全キーを持つかどうかは確認しない。「`attrs` 付きの `table` は `validate` が生成したものと信頼する」実装判断であり、手動で壊れた `table` を渡した場合は素通しし得る。

### 1-3. `_resolve_meta` を削除する

`_resolve_meta` を完全に削除する。その役割は `validate` 内部呼び出し + `detach` に置き換える。

### 1-4. `group`/`orderby`/`expand` の冒頭パターンを変更する

```perl
# 変更前
my ($rows, $meta, $attrs, $order) = _resolve_meta($table);

# 変更後
my $table = validate($aoh);          # rows でも table でも受け取れる
return $table unless @$table;        # 空テーブルはそのまま返す
my ($rows, $meta) = detach($table);
my $attrs = $meta->{'#'}{attrs};
```

各関数の引数名も `$table` → `$aoh` に変更する。

### 1-5. `orderby` のシグネチャを変更する

`@cols`（フラットなリスト）を `$cols`（配列リファレンス）に変更する。

```perl
# 変更前
sub orderby {
    my ($table, @cols) = @_;
    ...
    _check_cols($attrs, @cols);
    my @sorted = sort {
        for my $col (@cols) { ... }
    } @$rows;
}

# 変更後
sub orderby {
    my ($aoh, $cols) = @_;
    ...
    _check_cols($attrs, @$cols);
    my @sorted = sort {
        for my $col (@$cols) { ... }
    } @$rows;
}
```

## 2. `docs/spec.md` の変更

以下に挙げる節をそれぞれ差し替える。旧前提（`group`/`orderby`/`expand` は validate 済み table のみ受け付ける、`_resolve_meta` による入口統一、`@cols` 記法）が残らないようにすること。

### 2-1. 「用語と記号」節

`@cols` の記号定義を削除する。

```
削除: - `@cols`: カラム名の並び
```

`$cols` の定義はすでに存在するため追加不要。

### 2-2. 「前提条件・制限事項」節

節全体を以下の内容に差し替える。

- `detach()` と `attach()` を除く各 API は内部で `validate()` を呼ぶ
- `attach()` は `validate()` を呼ばない低レベルプリミティブ
- `detach()` と `attach()` を除く各 API は `rows` と `table` のどちらを受け取ってもよい
- `attrs` 付き `table` を `$cols` なしで `validate()` に渡すと、各行チェックをスキップして同一参照を返す
- 上記のアーリーリターンは「先頭行に `attrs` がある = validate 済みと信頼する」実装判断に基づく
- 行ごとにキー集合が異なる場合の動作は保証されない。処理中に存在しないカラムが見つかった場合は `die` する

サンプルコードを以下に置き換える（事前 validate 不要）：
```perl
my $sorted  = orderby(\@rows, ['A', 'B']);
my $grouped = group(\@rows, ['A']);
my $flat    = expand($grouped);
```

### 2-3. 「データ構造 — `rows` と `table`」節

以下の記述を削除または更新する。

削除：
```
`group`・`expand`・`orderby` は `table` のみ受け付ける。
```

更新後：
```
`detach()` と `attach()` を除く各 API は `rows` と `table` のどちらも受け付ける。
`validate` の戻り値は常に `table`。メタデータのない形が必要な場合は `detach` を利用する。
```

### 2-4. 「コーディング方針」節

節全体を以下の内容に差し替える。

- `detach()` と `attach()` を除く各パブリック API は冒頭で `validate($aoh)` を呼び出す
- `validate` の戻り値を `detach` で分離してメタとデータ行を取り出す
- 本体ロジックはメタデータを知らない内部関数に切り出す
- 処理完了後、トップレベルでのみ `attach` でメタデータを先頭に戻す
- `attach` は `validate` を呼ばず、渡された `$rows` と `$meta` をそのまま組み立てる
- `_resolve_meta` は廃止する（validate + detach に置き換え）

### 2-5. 「関数仕様 — `validate`」節

節全体を `$cols` なし / `$cols` あり の2ブロックに明確に分けて書き直す。各ブロックで rows 入力と table 入力の扱いを読めるようにする。

**`$cols` なし のブロック：**

| 入力形式 | アーリーリターン | 検証 | 結果 |
|---|---|---|---|
| `rows` | なし | あり（型推論） | `attrs` のみ付きの `table` を返す。`order` は生成しない |
| `table`（`attrs` あり） | あり | なし | 同一参照をそのまま返す。型の不整合があっても再検証しない |
| 空テーブル | なし | なし | `[]` をそのまま返す |

補足：`table` 入力のアーリーリターンは「先頭行に `attrs` がある = validate 済みと信頼する」実装判断に基づく。
各行の型整合は確認しない。旧仕様の「既存 `attrs` が `num` のカラムに非数値 → die」は、
`table` 入力＋`$cols` なしの場合はアーリーリターンにより **die しなくなる**（動作変更）。

**`$cols` あり のブロック：**

| 入力形式 | アーリーリターン | 検証 | 結果 |
|---|---|---|---|
| `rows` | なし | あり（`$cols` キー集合照合 + 型推論） | `attrs` + `order` 付きの `table` を返す |
| `table`（`attrs` あり） | なし | あり（`$cols` と `attrs` キー集合照合） | `$cols` の順序で `order` を設定した `table` を返す |
| 空テーブル | なし | なし | `[]` をそのまま返す |

### 2-6. 「関数仕様 — `group`」節

以下を更新する。

| 変更箇所 | 変更内容 |
|---|---|
| 引数の `$table` | `$aoh`（`rows` / `table` どちらでもよい）に変更 |
| 手順1 | `_resolve_meta でメタデータとデータ行を取り出す（attrs がなければ die）` → `validate($aoh) を呼び出し、detach でメタとデータ行を取り出す` |

### 2-7. 「関数仕様 — `orderby`」節

以下を更新する。

| 変更箇所 | 変更内容 |
|---|---|
| 引数の `$table` | `$aoh`（`rows` / `table` どちらでもよい）に変更 |
| 引数の `@cols` | `$cols`（カラム名の配列リファレンス）に変更 |
| 手順1 | `_resolve_meta でメタデータとデータ行を取り出す（attrs がなければ die）` → `validate($aoh) を呼び出し、detach でメタとデータ行を取り出す` |

| 呼び出し例 | `orderby($table, 'A', 'B')` → `orderby($aoh, ['A', 'B'])` |

### 2-8. 「関数仕様 — `expand`」節

以下を更新する。

| 変更箇所 | 変更内容 |
|---|---|
| 引数の `$table` | `$aoh`（`rows` / `table` どちらでもよい）に変更 |
| 手順1 | `_resolve_meta でメタデータとデータ行を取り出す（attrs がなければ die）` → `validate($aoh) を呼び出し、detach でメタとデータ行を取り出す` |

### 2-9. 「エラーハンドリング」節

以下の行を削除する（validate が内部で呼ばれるため "validate 未実行" シナリオは発生しない）：

削除: group: attrs なし（validate 未実行）または未知カラム指定時に die

## 4. `test/tabletools.t` の変更

### 4-1. `validate` テストを `$cols` なし / `$cols` あり × rows / table のマトリクスに整理し直す

単発追加ではなく、test-spec.md のマトリクス（A-1〜A-7、B-1〜B-7）に対応するテストを揃える方針で更新する。
既存テストの取り扱いは以下のとおり。

| 旧テスト名 | 対応マトリクス | 対応方針 |
|---|---|---|
| `cols なし・正常` | A-1 | そのまま存続 |
| `cols なし・キー不一致で die` | A-2 | そのまま存続 |
| `cols なし・空テーブル` | A-3 | そのまま存続 |
| `cols なし・既存 order を保持` | A-5 | そのまま存続（内容はアーリーリターンの確認に統合） |
| `cols なし・既存 attrs が num のカラムに非数値で die` | A-6 | **動作変更のため書き換え**（die → 同一参照が返る） |
| `cols なし・既存 attrs が str のカラムに数値は通過` | A-6 に統合 | 削除（アーリーリターンで自明） |
| `cols あり・メタデータ付きで返す` | B-1 | そのまま存続 |
| `cols あり・空テーブル` | B-3 | そのまま存続 |
| `cols あり・キー不一致で die` | B-2 | そのまま存続 |

| `cols あり・既存 order と異なる順序で order が上書きされる` | B-6 | そのまま存続 |

**新規追加するテスト：**

```perl
# A-4: $cols なし・attrs 付き table → 同一参照（アーリーリターン）
subtest 'validate: cols なし・attrs 付き table は同一参照が返る' => sub {
    my $table  = validate([{A => 1, B => 'x'}, {A => 2, B => 'y'}], ['A', 'B']);
    my $result = validate($table);
    is($result, $table, '同一参照が返る（アーリーリターン）');
};

# A-6: $cols なし・attrs が num のカラムに非数値値 → die しない（動作変更）
# 旧テスト 'cols なし・既存 attrs が num のカラムに非数値で die' を以下に書き換える
subtest 'validate: cols なし・attrs 付き table は再検証しない（die しない）' => sub {
    my $broken = [{'#' => {attrs => {A => 'num'}}}, {A => 'not_a_number'}];
    my $result = eval { validate($broken) };
    ok(!$@,             'die しない（アーリーリターン）');
    is($result, $broken, '同一参照が返る');
};

# B-4: $cols あり・table 入力・$cols と attrs が一致 → order が設定される
subtest 'validate: cols あり・attrs 付き table・$cols 順で order が設定される' => sub {
    my $table  = validate([{A => 1, B => 'x'}, {A => 2, B => 'y'}]);   # attrs のみ、order なし
    my $result = validate($table, ['B', 'A']);
    is_deeply($result->[0]{'#'}{order}, ['B', 'A'], '$cols 順で order が設定される');
};
```

### 4-2. `orderby` の全テストを `$cols` arrayref 形式に変更する

`tabletools.t` 内の全 `orderby` 呼び出しを以下の形式に統一する。

```perl
# 変更前（全箇所）
orderby($table, 'A')
orderby($table, 'B')
orderby($table, 'A', 'B')

# 変更後（全箇所）
orderby($table, ['A'])
orderby($table, ['B'])
orderby($table, ['A', 'B'])
```

### 4-3. `group`/`orderby`/`expand` の "バリデートなしで die" テストを置き換える

**`group`：**
```perl
# 変更前: 'group: バリデートなしの AoH で die'
subtest 'group: rows を直接渡せる' => sub {
    my $rows    = [{A => 1, B => 'x'}, {A => 1, B => 'y'}, {A => 2, B => 'z'}];
    my $grouped = group($rows, ['A']);
    is($grouped->[1]{A}, 1, 'rows を直接渡してもグループ化される');
    is(scalar @{$grouped->[1]{'@'}}, 2, 'A=1 の子は2件');
};
```

**`orderby`：**
```perl
# 変更前: 'orderby: バリデートなしの AoH で die'
subtest 'orderby: rows を直接渡せる' => sub {
    my $rows   = [{A => 3}, {A => 1}, {A => 2}];
    my $sorted = orderby($rows, ['A']);
    is($sorted->[1]{A}, 1, '1行目 A=1');
    is($sorted->[2]{A}, 2, '2行目 A=2');
    is($sorted->[3]{A}, 3, '3行目 A=3');
};
```

**`expand`：**
```perl
# 変更前: 'expand: バリデートなしの AoH で die'
subtest 'expand: rows を直接渡せる' => sub {
    my $rows   = [{A => 1, B => 'x'}, {A => 2, B => 'y'}];
    my $result = expand($rows);
    is(scalar @$result, 3, 'meta + 2行 = 3要素');
    is($result->[1]{A}, 1, '1行目 A=1');
    is($result->[2]{A}, 2, '2行目 A=2');
};
```

### 4-4. `attach` に `$meta` 同一参照テストを追加する
