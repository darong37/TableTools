# TableTools 設計ドキュメント

作成日: 2026-04-02

## 概要

`TableTools` は Perl の Array of Hashes（AoH）を操作するためのユーティリティモジュール。
データベースのテーブルに相当する構造（同じキー集合を持つハッシュリファレンスの配列）を
検証・グループ化・展開するための関数群を提供する。

## モジュール構成

`Exporter` ベースの純粋な関数モジュール。状態なし、OO なし。

```perl
use TableTools qw(validate group expand detach attach);
```

## 前提条件・制限事項

`group` / `expand` / `detach` / `attach` は、**`validate` を通過したテーブル**（全行が同じキー集合を持つ）を前提とする。

```perl
# 正しい使い方
my $table = validate(\@rows, ['A', 'B', 'C']);
my $grouped = group($table, ['A']);
```

`validate` を通過していないテーブルを渡した場合、行ごとにキー集合が異なると動作は保証されない。ただし、処理中にアクセスしようとしたカラムが存在しない行が見つかった場合は `die` する。

## データ構造

### テーブル（`$table`）

すべての関数が受け付ける基本型。メタデータ付き・なしどちらも `$table` として扱う。

```perl
# 純粋な AoH（メタデータなし）
[
    {A => 1, B => 'foo', C => 3},
    {A => 5, B => 'bar', C => 7},
]

# メタデータ付き AoH
[
    {'#' => [{col => 'A', attr => 'num'}, {col => 'B', attr => 'str'}, {col => 'C', attr => 'num'}]},
    {A => 1, B => 'foo', C => 3},
    {A => 5, B => 'bar', C => 7},
]
```

### メタデータ行

`'#'` キーのみを持つハッシュリファレンス。テーブルの先頭に置く。

```perl
{
    '#' => [
        {col => 'A', attr => 'num'},
        {col => 'B', attr => 'str'},
        {col => 'C', attr => 'num'},
    ]
}
```

- `col`: カラム名
- `attr`: 型情報。`'num'`（全値が数値）または `'str'`（1つでも文字列を含む）
- 配列の順序がカラムの順序を表す

**規約:**
- `'#'` キーを持つ行はデータ行として処理しない
- メタデータ行が存在すれば各関数はこれを読み取って活用する
- メタデータ行がなくても全関数は正常に動作する（カラム順序は不定）
- `validate($table, $cols)` はメタデータ付きテーブルを返す。`validate($table)` はメタデータを付加せず入力をそのまま返す
- `group` / `expand` は入力にメタデータ行があれば出力にも引き継ぐ

### グループ化後のテーブル

子行は `'@'` キーに配列リファレンスとして格納する。

```perl
[
    {'#' => [{col => 'A', attr => 'num'}, ...]},
    {A => 1, '@' => [
        {B => 'foo', C => 3},
        {B => 'bar', C => 7},
    ]},
    {A => 5, '@' => [
        {B => 'baz', C => 9},
    ]},
]
```

`'@'` で始まるキーは子行の配列リファレンスを表す規約。

## 関数仕様

### `validate($table, $cols)`

テーブルを検証する。`$cols` の有無によって戻り値が異なる。

**引数:**
- `$table`: AoH（メタデータあり・なし問わず）
- `$cols`: カラム名の配列リファレンス（省略可）。例: `['A', 'B', 'C']`

**処理:**

`$cols` を指定した場合:
1. 全データ行が `$cols` と同じキー集合を持つか検証
2. 各カラムの型を推論（全値が数値 → `'num'`、1つでも文字列 → `'str'`）
3. `$cols` の順序でメタデータ行を先頭に付加したテーブルを返す

`$cols` を省略した場合:
1. 最初のデータ行のキーを基準カラム集合として取得
2. 全データ行が同じキー集合を持つか検証
3. 純粋な AoH をそのまま返す（メタデータは付加しない）

**戻り値:**
- `$cols` あり: メタデータ付き `$table`（`scalar @$table` はデータ行数 + 1）
- `$cols` なし: 純粋な `$table`（入力をそのまま返す）

**エラー:** 検証失敗時は `die`

```perl
my $table = validate(\@rows, ['A', 'B', 'C']);  # メタデータ付き AoH を返す
my $table = validate(\@rows);                    # 検証のみ、純粋 AoH を返す
```

### `group($table, @cols_list)`

テーブルを多段グループ化する。

**引数:**
- `$table`: AoH（メタデータあり・なし問わず）
- `@cols_list`: グループ化するカラム名の配列リファレンスのリスト

**処理:**
1. メタデータ行を退避
2. 型情報を取得（メタデータがあれば `'#'` から、なければ内部関数 `_attrs` で全行スキャン）
3. 全グループキーの順（`@cols_list` の各配列を先頭から順に並べた順序、例: `['A'],['B','C']` なら `A, B, C` の順）でデータ行をソート
   - `attr` が `'num'` のカラムは `<=>` で比較
   - `'str'` のカラムは `cmp` で比較
3. 先頭の配列リファレンスのカラムでグループ化し、子行を `'@'` に格納
4. 子行に対して残りの配列リファレンスで再帰的にグループ化
5. メタデータ行を先頭に付加して返す

**戻り値:** グループ化された `$table`

```perl
# 2段グループ化
my $grouped = group($table, ['A'], ['B', 'C']);

# 結果イメージ
# [
#   {'#' => [...]},
#   {A => 1, '@' => [
#       {B => 'foo', C => 3, '@' => [{D => 10}, {D => 20}]},
#   ]},
# ]
```

### `expand($table)`

グループ化されたテーブルを完全にフラット化する。何重のネストでも一度にフラット化する。

**引数:**
- `$table`: AoH（グループ化済み・純粋どちらでも）

**処理:**
1. メタデータ行を退避
2. 全データ行を再帰的に展開（`'@'` キーを持つ行は子行と親のキーをマージしてフラット化）
3. メタデータ行を先頭に付加して返す

**戻り値:** フラットな `$table`

```perl
my $flat = expand($grouped);
```

### `detach($table)`

テーブルからメタデータ行を分離する。

**引数:**
- `$table`: AoH

**戻り値:** `($meta, $bare_table)`
- `$meta`: メタデータハッシュリファレンス、またはメタデータなしの場合は `undef`
- `$bare_table`: メタデータ行を含まない AoH

```perl
my ($meta, $bare_table) = detach($table);
```

### `attach($table, $meta)`

メタデータ行をテーブルの先頭に付加する。

**引数:**
- `$table`: AoH
- `$meta`: メタデータハッシュリファレンス（`undef` の場合は何もしない）

**戻り値:** `$table`（`$meta` がある場合はメタデータ行を先頭に付加したもの、`undef` の場合は入力をそのまま返す）

```perl
my $table = attach($bare_table, $meta);
```

## 型推論ルール

`validate`（`$cols` あり時）および内部関数 `_attrs` が使用する。

- `Scalar::Util::looks_like_number` で全値を検査
- カラムの全値が数値 → `attr => 'num'`（ソートに `<=>` を使用）
- 1つでも数値でない値がある → `attr => 'str'`（ソートに `cmp` を使用）

## 内部関数

### `_attrs($table)`

エクスポートしない内部関数。全データ行をスキャンして各カラムの型情報を返す。
`validate` および `group` の両方から利用する共通ユーティリティ。

**利用箇所:**
- `validate($table, $cols)`: `_attrs` で型情報を取得し、`$cols` の順序に従ってメタデータ行を構築する
- `group($table, ...)`: メタデータなし時に `_attrs` で型情報を取得し、ソートの比較方式を決定する

**戻り値:** カラム名をキー、`'num'` または `'str'` を値とするハッシュリファレンス

```perl
# 内部利用イメージ
my $attrs = _attrs($table);
# { A => 'num', B => 'str', C => 'num' }

# validate での使い方（概念）
my $attrs = _attrs($table);
my $meta  = {'#' => [map { {col => $_, attr => $attrs->{$_}} } @$cols]};
```

## エラーハンドリング

- `validate`: キー集合の不一致時に `die`
- `group` / `expand` / `detach` / `attach`: 入力が AoH でない場合に `die`

## コーディング方針

各関数のメタデータ処理は「先頭で一度だけ行う」という方針に統一する。

- **関数の入口でメタデータを分離する**: 処理開始前に `detach` でメタデータ行を取り出し、以降の本体ロジックは純粋な AoH（`$rows`）だけを対象に進める
- **本体ロジックはメタデータを知らない**: ソート・グループ化・展開などの処理は、メタデータの有無を意識せず純粋な AoH だけを扱う内部関数に切り出す
- **関数の出口でメタデータを付け直す**: 処理完了後、トップレベルでのみ `attach` でメタデータを先頭に戻す
- **再帰・中間処理にはメタデータを持ち込まない**: 再帰呼び出しや中間処理の内部でメタデータの分離・付加を繰り返してはならない

この方針により、各関数の責務が明確になり（「メタデータの入出力」と「AoH の操作」が分離される）、再帰処理などの本体ロジックがシンプルに保たれる。

## 設計上の判断

- **なぜ Exporter か**: 操作対象が単純な AoH であり状態を持つ必要がない。OO 化は過剰。
- **なぜ `'#'` キーか**: Perl のコメント記号（`#`）と同じ直感。「データではなく注釈」であることが一目でわかる。データキーと衝突しにくい。
- **なぜ `'@'` キーか**: Perl の配列シジル（`@`）と同じ直感。「このキーの値は配列リファレンス（子行の集合）」であることが一目でわかる。
- **メタデータは任意**: 純粋な AoH でも全関数が動作する。メタデータは `validate($table, $cols)` によって新規付加される。`group` / `expand` は入力にメタデータがあれば出力にも引き継ぎ、なければなしのまま返す。`validate($table)` は検証のみでメタデータを付加しない。
- **`scalar @$table` の off-by-1**: メタデータ行を含むため既知の trade-off。データ行数は `scalar @$table - 1`。
- **子配列にメタデータは持たない**: カラム順序はトップレベルの全カラムを表すものであり、ネスト先には不要。
