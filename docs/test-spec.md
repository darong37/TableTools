# TableTools テスト仕様書

## 用語

- `AoH`: ハッシュリファレンスの配列リファレンス
- `rows`: メタデータを持たない AoH
- `table`: 先頭行にメタデータを持つ AoH

## `detach`

| # | テストケース | 期待結果 |
|---|---|---|
| 1 | メタデータ行ありの `table` | `($rows, $meta)` を返す。`$rows` はデータ行のみの配列リファレンス、`$meta` はメタデータハッシュ |
| 2 | メタデータ行なしの `rows` | `($rows, undef)` を返す |

## `attach`

| # | テストケース | 期待結果 |
|---|---|---|
| 1 | `$meta` あり | メタデータ行を先頭に付加した `table` を返す |
| 2 | `$meta` が `undef` | 入力をそのまま返す |
| 3 | 渡した `$meta` がそのまま先頭に付く | `$table->[0]` が渡した `$meta` と同一参照である |

## `validate`

### `$cols` なし

| # | テストケース | 期待結果 |
|---|---|---|
| A-1 | `rows` 入力・全行キー一致 | `attrs` のみのメタデータ行を先頭に付加した `table` を返す。`order` は新たに生成しない |
| A-2 | `rows` 入力・キー不一致の行あり | `die`（メッセージに "column" を含む） |
| A-3 | 空テーブル | `[]` をそのまま返す（メタデータを生成しない） |
| A-4 | `attrs` 付き `table` 入力（アーリーリターン） | 同一参照をそのまま返す（再検証しない） |
| A-5 | `order` 付き `table` 入力 | 既存 `order` を保持したまま返す |
| A-6 | `attrs` 付き `table` 入力（`num` カラムに非数値があっても通過） | アーリーリターンにより同一参照を返す（再検証しない） |
| A-7 | `rows` 入力・型推論（num/str） | `num` のみのカラムは `attrs` が `'num'`、非数値を含むカラムは `'str'` になる |
| A-8 | `rows` 入力・`undef` を含む | `undef` は空文字 `''` に置き換えられる |
| A-9 | `attrs` 付き `table` 入力・`undef` を含む | 同一参照のまま `undef` が空文字 `''` に置き換えられる |

### `$cols` あり

| # | テストケース | 期待結果 |
|---|---|---|
| B-1 | `rows` 入力・正常データ | `attrs` と `order` 両方を含むメタデータ行を先頭に付加した `table` を返す。カラム順・型情報が `$cols` 指定順で正しく付与される |
| B-2 | `rows` 入力・キー不一致の行あり | `die`（メッセージに "column" を含む） |
| B-3 | 空テーブル | `[]` をそのまま返す（メタデータを生成しない） |
| B-4 | `attrs` 付き `table` 入力 | `$cols` 順で `order` が設定された `table` を返す |
| B-5 | `attrs` との集合不一致 | `die`（`$cols` と `attrs` のキー集合が一致しない場合） |
| B-6 | 既存 `order` と異なる順序 | `$cols` の順序で `order` が上書きされる |
| B-7 | `rows` 入力・確定済み `num` カラムに非数値 | `die` |
| B-8 | `rows` 入力・`undef` を含む | `undef` は空文字 `''` に置き換えられ、その後 `attrs` と `order` が付く |

## `group`

| # | テストケース | 期待結果 |
|---|---|---|
| 1 | 1段グループ化（`validate` 済み `table`） | グループキーの値でまとめられた行を `'@'` に格納。子行からはグループキーのカラムが除かれる。入力のメタデータが引き継がれる |
| 2 | ソート済み入力で正常動作 | 入力行の順番どおりにグループが並ぶ（内部ソートなし） |
| 3 | 順序違反（同一キーが非連続で再出現）で die | `die`（メッセージに "out of order" を含む） |
| 4 | `attrs`・`order` 付き `table` のグループ化 | 入力の `attrs`・`order` がそのまま引き継がれる |
| 5 | 2段グループ化 | 先頭の `@cols_list` でグループ化し、子行を残りの `@cols_list` で再帰的にグループ化する |
| 6 | 存在しないカラム指定 | `die`（メッセージに "column" を含む） |
| 7 | `rows` を直接渡せる | `rows`（メタデータなし）を渡しても正常にグループ化できる |
| 8 | 空の `rows` を渡す | `[]` を返す |
| 9 | グループ指定なし | `validate` 済みの結果をそのまま返す。`rows` 入力なら `table`、空なら `[]` |

| 8 | 空の `$cols` を渡す | `validate` 済みの結果をそのまま返す。`rows` 入力なら `table`、空なら `[]` |
